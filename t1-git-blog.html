<<<<<<< HEAD Version Control? Git? GitHub? Why These Important Development Tools Aren’t As Intimidating As They May Seem

By Courtney Wiggins August 2, 2015

The world of programming can be very daunting for most, especially beginners. There are fancy technical terms, methods, and best practices to consider that can confuse even the most well studied student or autodidactic. It is easy to confuse what git is, How Version Control Works, and why GitHub is not git. The beginner is left feeling lost and question how important these three resources are. In reality, Version Control, git, and GitHub are beneficial and makes producing code that much more efficient.

Version Control tracks all of the modifications you have made to a file or set of files over a set period so that it is easier for you to revert back to previous copies of your file later on. This quality is great since it provides a ‘safety net’ for the user. For example, let’s say that the user unfortunately deleted a version of their file/work. Using Version Control, this user could then go back and retrieve a copy of the lost file in its state prior to it becoming deleted. This saves the user much heartache and allows them to continue work on their project.

Git is a type of version control that stores data locally. Git allows you to ‘commit’, ‘push’, and ‘pull’ your work on your computer. Each and every time you go to submit changes you have made to a file or files through git, it takes a snapshot of what your file(s) look like at that moment in time. It is then stored as a reference point for when you need to refer back to it later. =======
